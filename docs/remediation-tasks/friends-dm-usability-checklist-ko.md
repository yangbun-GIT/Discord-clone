# 친구/DM 사용성 보강 확인 문서

## 문서 목적

이 문서는 사용자가 다음 구현 범위를 빠르게 확인하기 위한 한국어 문서입니다.
Codex가 실제 구현할 때는 영어 작업 문서인
`docs/remediation-tasks/friends-dm-usability-implementation-reference.md`와
기존 개발 문서 `docs/remediation-tasks/friends-home-remediation-2026-06-20.md`를
함께 참조합니다.

현재 목표는 단순히 화면을 더 예쁘게 꾸미는 것이 아닙니다. 실제 Discord 클론으로
사용하기 불편한 부분, 기능처럼 보이지만 동작하지 않는 부분, 피드백이 부족해
상태를 알기 어려운 부분을 구현 대상으로 잡는 것입니다.

## 구현 대상에서 제외하면 안 되는 항목

아래 항목은 “아직 구현이 안 되어 있으니 숨긴다”가 최종 결론이 아닙니다. 지금은
가짜 버튼을 없애기 위해 임시로 숨길 수 있지만, Discord 클론 완성도를 위해 실제
구현해야 하는 항목입니다.

1. 친구 프로필 보기
   - 친구 목록, 활동 패널, 우클릭 메뉴에서 친구 프로필을 볼 수 있어야 합니다.
   - 프로필에는 이름, 핸들, 상태, 활동, 메시지 보내기, 친구 관리 액션이 포함되어야 합니다.

2. 친구/DM 통화 시작
   - 친구 화면이나 DM 화면에서 바로 통화를 시작할 수 있어야 합니다.
   - 현재 서버 음성 채널과 충돌하지 않도록 별도 DM 통화 흐름이 필요합니다.

3. 대화 알림 끄기
   - 특정 DM을 음소거하면 메시지는 계속 받되 일반적인 안 읽음/알림 강조는 줄어야 합니다.

4. 새 DM 시작
   - 좌측 DM 목록의 `+` 버튼과 “새 다이렉트 메시지 시작”은 실제 친구 선택 창을 열어야 합니다.
   - 친구를 선택하면 기존 DM을 열거나 새 DM을 만들어야 합니다.

5. 친구/DM 우클릭 메뉴
   - 우클릭 메뉴는 실제 클릭한 친구나 DM을 대상으로 동작해야 합니다.
   - 대상이 불명확한 상태에서 가짜 메뉴를 보여주면 안 됩니다.

6. 친구 요청 도착 피드백
   - 다른 사용자가 친구 요청을 보내면 받는 쪽에서 즉시 알 수 있어야 합니다.
   - 친구 요청 탭 숫자, `@me` 배지, 앱 내부 알림 등으로 확인 가능해야 합니다.
   - 수락/거절 후에는 피드백이 정리되어야 합니다.

7. 채팅 메시지 하단 기준 표시
   - DM과 서버 텍스트 채팅은 최신 메시지가 기본적으로 하단에 보여야 합니다.
   - 대화가 길어져도 사용자가 매번 아래로 스크롤하지 않아야 합니다.
   - 사용자가 과거 메시지를 보고 있을 때는 위치를 유지하고, 새 메시지가 오면
     “최신 메시지로 이동” 같은 작은 버튼을 제공하는 방식이 적절합니다.

8. DM 헤더와 우측 프로필 패널 액션
   - DM 화면의 통화, 프로필, 검색, 멤버 정보 같은 버튼은 실제 선택된 DM을 대상으로 동작해야 합니다.
   - 아직 구현하지 않을 버튼은 활성 버튼처럼 보이지 않게 해야 합니다.

## 현재 구현되어 있어 유지할 항목

아래 항목은 이미 실제 동작이 있으므로 회귀 테스트 대상입니다.

- 친구 요청 보내기
- 친구 요청 수락/거절/취소
- 친구 삭제
- 차단/차단 해제
- 친구에게 메시지 보내기
- DM 송수신
- 친구 상태 동기화
- 서버 텍스트 채팅
- 서버 음성 채널 통화
- Cloudflare 임시 URL 기반 특정 외부망 DM/음성 성공 기록

## 현재 구현 대상에서 제외 가능한 항목

아래 항목은 유용할 수 있지만 이번 친구/DM 사용성 보강의 핵심 범위는 아닙니다.

- Nitro, 상점, 퀘스트, 결제성 기능
- 연락처 가져오기나 외부 계정 연동
- 대규모 검색 인덱싱
- 상시 public VM/VPS 배포
- Discord의 모든 부가 기능을 완전 복제하는 작업

## 다음 구현 우선순위

1. 친구 요청 도착 피드백
   - 실시간 요청 도착을 바로 알아차릴 수 있어야 합니다.

2. 새 DM 시작
   - 좌측 `+`와 검색 팝오버의 “새 DM 시작”이 실제로 DM을 만들거나 열어야 합니다.

3. 메시지 하단 기준 표시
   - DM과 서버 채팅 모두 최신 메시지를 보기 쉽게 해야 합니다.

4. 친구 프로필 보기
   - 친구 정보를 확인하고 메시지/관리 액션으로 이어져야 합니다.

5. 대화 알림 끄기
   - DM별 음소거가 실제 안 읽음/알림 강조에 영향을 주어야 합니다.

6. 친구/DM 통화 시작
   - 친구나 DM에서 직접 통화로 진입할 수 있어야 합니다.

7. 우클릭 메뉴 복구
   - 각 메뉴가 정확한 대상을 알고 동작할 때만 다시 활성화해야 합니다.

8. DM 헤더/프로필 패널 액션 정리
   - 보이는 버튼은 실제 동작하거나, 아직 구현 전이면 명확히 정리해야 합니다.

## 확인 방법

- 두 계정으로 접속해 친구 요청을 보내고, 받는 쪽에서 새 요청 피드백이 뜨는지 확인합니다.
- 좌측 DM `+` 버튼으로 친구를 선택해 DM이 열리는지 확인합니다.
- 긴 DM이나 서버 채팅에서 새로고침/채널 전환 후 최신 메시지가 바로 보이는지 확인합니다.
- 친구 프로필 보기, 통화 시작, 대화 알림 끄기 버튼이 실제 동작하는지 확인합니다.
- 우클릭 메뉴가 클릭한 친구나 DM에 대해 정확히 동작하는지 확인합니다.
- 가짜 안내만 뜨는 활성 버튼이 남아 있지 않은지 확인합니다.
